The 6 Phases We Built

Phase 1

Foundation

  • Companies register and get an API key
  • Every request is authenticated
  • Events are accepted and stored safely
  • Health check endpoint
Phase 2

Queue & Worker

  • Events drop into a job queue instantly
  • A background worker picks them up
  • Rules are stored and cached in Redis
  • Retries on failure — 3 attempts, exponential backoff
Phase 3

Rule Engine

  • Worker evaluates each rule condition
  • 12 operators — equals, greater than, contains, regex…
  • Supports nested fields like order.address.city
  • Every match is logged as pending
Phase 4

Action Executors

  • Webhook fires an HTTP call to any URL
  • Email sends via SMTP with template variables
  • Slack posts a message to your channel
  • Every attempt logged — success or failed
Phase 5

Reliability

  • Rate limiter — per-tenant sliding window via Redis
  • Dead Letter Queue for exhausted jobs
  • Plan-based limits — free / pro / enterprise
  • Duplicate job guard — no double-processing
Phase 6

Observability & Polish

  • GET /logs — paginated audit log with cursor
  • GET /stats — success rate, top rules, totals
  • Swagger UI at /docs — try every endpoint live
  • Docker, Fly.io deploy, Postman collection, README
